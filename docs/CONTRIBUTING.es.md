Guía de contribución al proyecto **Bleach: Reiryoku**

## Resumen

En este documento se describe el proceso para contribuir al proyecto.

## Código de conducta

Lo primero es leer y seguir nuestro [Código de conducta](CODE_OF_CONDUCT.es.md).

## Tipos de contribución

Hay dos formas principales: reportar problemas (issues) y enviar cambios (pull requests). Ambas se describen a continuación.

## Issues

Puedes ayudar creando issues en el repositorio:

- **Informes de errores** — si encuentras un bug en el mod (objetos, crafteo, balance, etc.)
- **Solicitudes de funciones** — si crees que al mod le falta alguna mecánica o tienes una idea de contenido

Al abrir un issue, describe el problema o la idea con claridad. Para consultas rápidas también puedes escribirnos en [Discord](https://discord.gg/7My4sVvafk).

## Pull requests

También puedes contribuir con pull requests. El proceso:

- Decidir qué quieres implementar o mejorar
- Hacer un fork del repositorio
- Crear una rama (p. ej. `feature/tu-funcionalidad` o `fix/tu-correccion`)
- Hacer y probar los cambios en Hytale
- Hacer commits con mensajes claros
- Abrir un pull request
- Esperar la revisión; podemos pedir ajustes antes de fusionar

Cuando se envía un pull request, al menos un miembro del proyecto lo revisa. Tras los cambios solicitados, el PR puede fusionarse.

## ¿Necesitas ayuda?

Únete a nuestro [Discord](https://discord.gg/7My4sVvafk) para novedades, oportunidades de prueba y hablar de contribuciones.
