Guia de contribuição do projeto **Bleach: Reiryoku**

## Visão geral

Este documento descreve o processo de contribuição para o projeto.

## Código de Conduta

Em primeiro lugar, leia e siga nosso [Código de Conduta](CODE_OF_CONDUCT.pt-br.md).

## Tipos de contribuição

Há duas formas principais de contribuir: reportar problemas (issues) e enviar alterações (pull requests). Ambas estão descritas abaixo.

## Issues

Você pode ajudar criando issues no repositório:

- **Relatos de bugs** — se encontrar um bug no mod (itens, craft, balanceamento etc.)
- **Pedidos de funcionalidades** — se achar que falta alguma mecânica ou ideia de conteúdo

Ao abrir uma issue, descreva o problema ou a ideia de forma clara. Para dúvidas rápidas você também pode nos contactar no [Discord](https://discord.gg/7My4sVvafk).

## Pull requests

Você também pode enviar alterações via pull request. O processo:

- Decidir o que deseja implementar ou corrigir
- Fazer um fork do repositório
- Criar uma branch (ex.: `feature/sua-funcionalidade` ou `fix/seu-fix`)
- Fazer e testar as alterações no Hytale
- Fazer commits com mensagens claras
- Abrir um pull request
- Aguardar a revisão; podemos pedir ajustes antes de mesclar

Após o envio, pelo menos um membro do projeto fará a revisão. Depois dos ajustes necessários, o PR pode ser mesclado.

## Precisa de ajuda?

Junte-se ao nosso [Discord](https://discord.gg/7My4sVvafk) para novidades, oportunidades de teste e discussão sobre contribuições.
