Guide de contribution au projet **Bleach: Reiryoku**

## Aperçu

Ce document décrit comment contribuer au projet.

## Code de conduite

Commencez par lire et respecter notre [Code de conduite](CODE_OF_CONDUCT.fr.md).

## Types de contribution

Il y a deux façons principales de contribuer : signaler des problèmes (issues) et proposer des modifications (pull requests). Les deux sont décrites ci-dessous.

## Issues

Vous pouvez aider en créant des issues dans le dépôt :

- **Signaler des bugs** — si vous trouvez un bug dans le mod (objets, craft, équilibrage, etc.)
- **Demandes de fonctionnalités** — si vous pensez qu’il manque une mécanique ou avez une idée de contenu

En ouvrant une issue, décrivez le problème ou l’idée clairement. Pour des questions rapides, vous pouvez aussi nous joindre sur [Discord](https://discord.gg/7My4sVvafk).

## Pull requests

Vous pouvez aussi proposer des changements via pull request. Le processus :

- Choisir ce que vous voulez implémenter ou corriger
- Faire un fork du dépôt
- Créer une branche (ex. `feature/votre-fonctionnalite` ou `fix/votre-correction`)
- Faire et tester vos changements dans Hytale
- Faire des commits avec des messages clairs
- Ouvrir un pull request
- Attendre la relecture ; nous pouvons demander des ajustements avant la fusion

À la soumission d’un pull request, au moins un membre du projet fait une relecture. Après d’éventuelles modifications, le PR peut être fusionné.

## Besoin d’aide ?

Rejoignez notre [Discord](https://discord.gg/7My4sVvafk) pour les actualités, les opportunités de test et les échanges sur les contributions.
