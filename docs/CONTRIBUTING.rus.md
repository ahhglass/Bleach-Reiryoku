Руководство по внесению вклада в проект **Bleach: Reiryoku**

## Обзор

В этом документе описан процесс участия в разработке мода.

## Кодекс поведения

Для начала прочитайте и соблюдайте наш [Кодекс поведения](CODE_OF_CONDUCT.rus.md).

## Типы вклада

Есть два основных способа помочь: сообщать о проблемах (issues) и отправлять изменения (pull requests). Оба варианта описаны ниже.

## Issues (проблемы и идеи)

Вы можете помочь, создавая issues в репозитории:

- **Отчёты об ошибках** — если нашли баг в моде (предметы, крафт, баланс и т.д.)
- **Предложения по функциям** — если считаете, что моду не хватает какой-то механики или контента

При создании issue по возможности опишите проблему или идею чётко. Для быстрых вопросов можно написать в [Discord](https://discord.gg/7My4sVvafk).

## Pull requests (изменения в коде)

Вы также можете присылать изменения через pull request. Кратко:

- Определите, что хотите реализовать или исправить
- Сделайте форк репозитория
- Создайте ветку (например, `feature/ваша-фича` или `fix/ваш-фикс`)
- Внесите изменения и проверьте их в Hytale
- Сделайте коммиты с понятными сообщениями
- Откройте pull request
- Дождитесь ревью; по необходимости мы попросим доработки перед слиянием

После отправки pull request хотя бы один участник проекта проведёт ревью. После исправлений (если нужны) изменения могут быть приняты в проект.

## Нужна помощь?

Присоединяйтесь к нашему [Discord](https://discord.gg/7My4sVvafk) — там публикуются обновления, можно участвовать в тестировании и обсуждать вклад в проект.
