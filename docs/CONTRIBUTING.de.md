## Verhaltenskodex

## Arten von Beiträgen

Es gibt im Wesentlichen zwei Wege: Issues (Probleme melden) und Pull Requests (Änderungen vorschlagen). Beides wird unten beschrieben.

## Issues

Ihr könnt uns helfen, indem ihr Issues im Repository erstellt:

- **Bug-Reports** — wenn ihr einen Fehler im Mod findet (Items, Crafting, Balance usw.)
- **Feature-Anfragen** — wenn euch eine Funktion oder Idee für neuen Inhalt fehlt

Beschreibt Problem oder Idee bitte möglichst klar. Für kurze Fragen erreicht ihr uns auch auf [Discord](https://discord.gg/7My4sVvafk).

## Pull Requests

Ihr könnt auch Pull Requests einreichen. Ablauf:

- Festlegen, was ihr umsetzen oder verbessern wollt
- Repository forken
- Branch erstellen (z. B. `feature/eure-funktion` oder `fix/euer-fix`)
- Änderungen machen und in Hytale testen
- Commits mit klaren Nachrichten erstellen
- Pull Request öffnen
- Auf Review warten; bei Bedarf bitten wir um Anpassungen vor dem Merge

Nach Einreichung eines Pull Requests führt mindestens ein Teammitglied ein Review durch. Danach kann der PR ggf. nach Nachbesserungen übernommen werden.

## Fragen?

Kommt in unseren [Discord](https://discord.gg/7My4sVvafk) — dort gibt es Updates, Testmöglichkeiten und Austausch zu Beiträgen.
